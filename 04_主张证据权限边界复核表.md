## 1. 这张表的用途

每一个核心主张都必须同时具有：明确任务、正确分母、评价方案、来源工件、必要限定语和允许后果。投稿前应逐行签核，避免把不同阶段的高分串成单一“模型进步”，或把内部一致性写成外部正确性。

## 3. 五类主张的允许与禁止

### 3.1 Decision-trace 主张

**允许写：**

- 许多结构化系统优先保存 current/final state；本项目的 PDF 和台账包含 process fragments，但未形成完整、统一、可回放的 decision trace。
- 本研究重建 provenance-bearing trace fragments，并审计哪些 action、deliberation、rule-version 和 reason 元素不可恢复。
- prospective intake 的目标是版本化保存 request、feedback、revision、accepted evidence、rule、reason、override 与 later outcome。
- decision-trace deficit 是本案例揭示的中心瓶颈之一。

**禁止写：**

- “数据库完全没有原因”或“过去 40 年都没有 decision traces。”
- “本研究完整恢复了真实会议、讨论和人类 deliberation。”
- “LMM 可以从 final state 补写档案中未记录的原因。”
- “decision trace 等于最终标签、普通操作日志或模型解释。”
- “缺少 decision trace 已被证明是所有高监管 Agent 的唯一/最大问题。”

### 3.2 多模态清理管线

**允许写：**

- 输出设计保留 source file、page、quote、visual state、track 和 timing。
- 数值、证据与时序 guards 将可疑记录送入 quarantine。
- fixed 40-case set 对三个预定义字段显示内部回归稳定性。
- 内部 AI-agent 盲读和研究者裁决发现了来源指针错误和定义不稳定。

**禁止写：**

- “所有字段均由专家确认正确。”
- “每个字段都来自某一 PDF 页。”
- “多个 agent 一致证明了 ground truth。”
- “gpt 浮动 alias 的抽取可逐字确定性复现。”

### 3.3 Model A

**允许写：**

- 在当前 424 案内部 county-grouped DN 任务中具有非平凡排序能力。
- 用于安排 review queue，帮助稀缺人工先看更像 DN 的材料。
- 不进入 current quadrant formula。

**禁止写：**

- “83.7% 准确率。”
- “努力充分概率”或“批准概率”。
- “DN 阳性即操作方已尽合理努力。”
- “Model A 衡量了 effort trace 的完整性或 reasonable-effort adequacy。”
- “排序没有任何现实后果。”

### 3.4 Model B 与 anomaly

**允许写：**

- 近似 locked historical shortfall teacher，并提出 reference stratum。
- current request 的 proposed-stop shortfall 与所选历史 reference distribution 比较。
- tail 是 explanation-needed/escalation flag。

**禁止写：**

- “Model B 准确预测真实工程难度。”
- “Fig. 7–9 证明 Model B 部署后各档校准为 10%。”
- “354 个案例都是经确认批准的先例。”
- “历史 D1 和当前 D1 都是 proposed stop。”
- “tail 代表努力不足、违规或应拒绝。”
- “conditioned P90 已证明公平。”
- “Model B 是 geography profile”或“通过地理位置预测客观难度。”
- “TD、井龄、套管、水、煤层属于 geography features。”
- “Model A/B 分别识别 effort 与 geography 两个因果因素。”
- “工具特征加地理难度即可衡量 effort adequacy。”

### 3.5 Agent 架构

**允许写：**

- 当前原型可以生成 preliminary review package，并将 reviewer action 单独保存。
- prefill LMM 提出候选；reconciliation LMM 当前 advisory。
- Model A 排队；Model B 经 bin→tail 间接影响 preliminary route。
- 目标架构要求 reviewer acceptance before consequence。
- 目标架构把每次人工决定写入 versioned decision-trace ledger，作为未来验证数据的候选来源。

**禁止写：**

- “当前代码已经强制字段级 human acceptance。”
- “reconciliation 已 gate CK2 和 route。”
- “RAG 已验证为 precedent system。”
- “decision log 已形成 clean labels。”
- “当前 decision log 已是完整 decision trace 或真实 deliberation gold standard。”
- “当前 lane 不包含任何 approval recommendation。”
- “这是多个自治 agent 共同投票的审批系统。”
