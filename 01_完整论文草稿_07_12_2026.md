## ——宾夕法尼亚州油气井封堵审查档案的回顾性方法研究

## 摘要

### 背景

监管档案正在被越来越多地用于训练文档智能和决策支持系统。然而，历史案卷不是决策发生的瞬间：同一档案可能同时包含申请、机构往来、后续作业、批准条件和完工证明。若把这些时间层压成一张表，模型可以借用决策后信息预测历史结果；若把历史批准或最终封堵状态直接当作“努力充分性”真值，模型还会继承选择性标签、删失、地方实践和文书习惯。由此得到的模型可能具有不错的区分度，却准确地回答了错误的问题。

### 目的

本研究以宾夕法尼亚州油气井封堵审查档案为回顾性案例，研究五个问题：（1）多模态抽取、来源追踪、确定性防护和多代理复核能否生成可审计证据；（2）在真实群组外推下，证据、地理、模型类别和特征丰富度分别贡献了什么；（3）为什么历史 shortfall/difficulty 不能解释为 effort adequacy；（4）哪些 checklist 与 documented non-routine difficulty（DN）信号能够在冻结规则、内部盲读和双向审计后幸存；（5）G9、Path-B 和 retrospective routing 的失败应当如何限制系统权限。

### 方法

我们重建并审计了一条研究链，包括异构 PDF 的多模态读取、文件—页码—原文—视觉状态来源元组、requested/approved/certificate 三条深度与时间轨、数值/证据/时序防护、隔离队列、按 county 分组的交叉验证、标签和删失审计、版本控制下的协议冻结、独立 AI-agent 盲读、研究者裁决以及预先规定后果的失败门控。源文件处理范围包括 635 个含 PDF 案卷和约 1,700 份 PDF；冻结 shortfall 队列包含 631 案、29 个 county；DN 与 Model A 分析包含 424 案，Model B 的 teacher-available 分析包含 423 案。我们进一步审计 170 列 canonical schema、2,874 条带日期事件以及工具和补救方法的记录稀疏性。

### 结果

多模态管线形成 633 个唯一案级记录、10,856 条深度证据行和 436 条隔离记录。固定的 40 案针对性回归集中，三个预定义字段的 precision 与 recall 均为 1.0；该结果只证明已知失败模式的内部回归稳定性，不代表全语料准确率。冻结 county-grouped 评估中，非地理证据/协变量 HGB 的 AUC 为 0.740±0.009，地理先验为 0.625±0.006；更丰富的 v6 特征未解决性能与构念问题。早期 step-location 模型的 AUC 0.916 被重新判定为循环性结果，因为零参数“最深事件”规则已恢复 79/103 个标签。89 案删失敏感性分析使 shortfall-risk AUC 从 0.757 降至排除后的 0.742 和负向重标后的 0.689，说明该标签不能支持 effort-adequacy 解释。

424 案中虽然 423 案至少有一条带日期事件，但明确“尝试且失败”的记录仅占 6.1%，障碍深度占 14.6%，最大到达深度占 18.9%，细粒度工作记录 flags 几乎均低于 2.5%；93.15% 的事件没有工具 token。困难类型在申请叙述中的记录率通常比独立工作记录层高约 3–74 倍，这描述的是记录选择性，而不是实际努力。项目内部裁决得到 112/424 个 DN 阳性。Model A 是 DN prescreen HGB classifier，内部 county-grouped AUC 为 0.8366±0.1097，只用于人工队列排序；Model B 是 locked historical shortfall ensemble 的 HGB proxy regressor，一套五折 county-grouped 评估的 Spearman 相关为 0.418±0.047，精确分档一致率 35.5%、相差不超过一档 72.3%，用于选择经验参照档，并通过后续 tail flag 间接影响 preliminary route。两者共享 17 个 request-time-compatible 字段，但均没有审批权限。

G9 的名义主比较为 7/13 对 105/411，Fisher 双侧 p=0.0477，但两个预先冻结的敏感性条件失败；Path-B 组合 AUC 0.6841，未达到 0.764 门槛；回顾性路由把 7 个深度型拒绝中的 5 个送入快线。相应地，approval/refusal、历史结果态自动路由和 anomaly 的不利裁决权限均被关闭。

### 结论

这些回顾性结果支持一种“有效性先于准确率”的研究纪律，而不是一个自动审批器。多模态模型可以提出候选证据，Model A 可以分配先看谁，Model B 可以提出用哪一把历史尺子比较，确定性规则可以表达明确缺项和冲突；但是事实接受、规范解释和最终行动必须保留给能够核验来源的合格审查员。历史研究管线已经生成可回溯证据工件；当前原型能够生成内部排序/分组分数和 preliminary review package，但尚未强制完整的运行时 provenance 与字段接受闭环。其真实审查效用尚未验证，历史结果态档案也不足以单独支持努力充分性解释。下一阶段需要新的申请时点数据、独立领域专家参考标准、operator/time/template 外部留出，以及前瞻性人机协同试验。

## 1. 引言

高后果文档智能的核心难题不是“能否从 PDF 里抽出一个数字”，而是“这个数字是什么、何时可知、由什么证据支持、可以授权系统做什么”。在监管审查中，这四个问题通常被历史档案的便利性遮蔽。完工档案看起来信息丰富，但它混合了申请时未知的未来事实；历史批准看起来像标签，但它可能只反映当时被观察和记录的部分世界；复杂模型看起来比简单模型更强，但它可能只是更擅长利用模板、地理或结果态线索。

宾夕法尼亚州油气井封堵档案提供了一个典型场景。申请人可能请求在井总深度（total depth, TD）以上停止作业，理由包括坍塌、硬填充物、套管失效、矿层或其他障碍。监管者真正关心的并不是历史上“最后堵到了哪里”这一孤立结果，而是：在申请当时，是否存在可核验的困难证据、做过什么、为何停止、还缺少什么信息，以及与可比情形相比是否需要进一步审查。

本项目经历了三次问题定义纠错。第一，早期“在批准案内部定位哪个步骤成为最终步骤”的任务，并不是预测监管决定；它后来被证明主要由最深事件这一标签定义本身驱动。第二，case-level shortfall 风险或难度并不等于 operator 的努力充分性。第三，结果态案卷无法可靠重建申请时点的信息集合。一个重要的子纠错是：文档中没有正式 effort narrative，不能推断“没有努力”；项目因而把构念收缩为 documented non-routine difficulty（DN）。

这段演化导向本文的核心命题：

> **准确率只有在测量、时间、构念、外推和决策权限五重有效性同时成立时，才具有高后果决策含义。**

本文作出四项贡献。

第一，我们给出一条以来源为中心的多模态证据工程管线，使结构化输出能够回到 PDF 页或 legacy register，并把不确定、冲突和时间污染显式隔离。第二，我们通过泄漏 ladder、零参数基线、county-grouped 评估、89 案删失审计和三个失败门控，展示“高分属于哪个任务”必须先于“分数有多高”。第三，我们提出 Model A/Model B 的受限分工：A 只排序人工注意力，B 只选择经验参照尺，二者不合成审批器。第四，我们提出一种按权限而非按模型数量组织的混合架构：LMM 负责候选证据和对账，deterministic rules 负责可见条件，人类审查员接受事实并承担最终行动。

## 2. 研究场景、数据范围与研究问题

### 2.1 数据并非一个统一分母

本项目在不同阶段使用了不同范围，不能把所有数字写成同一个 cohort。

**表 1. 数据范围及其分析用途**

| 含 PDF 案卷 | 635 | 可进行全文档抽取的源文件范围，约 1,700 份 PDF |
| 无本地 PDF 来源目录 | 277 | 不得解释为负证据；可能构成数字可得性选择机制 |

### 2.2 五个研究问题

- **RQ1（证据有效性）**：多模态读取、来源追踪、确定性防护和内部多代理复核能否产生可审计的结构化证据？
- **RQ2（诚实泛化）**：在 county-grouped 外推下，非地理证据/协变量、地理先验、模型类别和特征丰富度分别对应怎样的内部表现？
- **RQ3（目标有效性）**：为什么 historical shortfall/difficulty 不能解释为 effort adequacy；选择性标签和删失如何改变结论？
- **RQ4（构念幸存）**：哪些 request checklist 和 DN 信号能在规则冻结、内部 AI-agent 盲读、来源裁决和双向审计后保留？
- **RQ5（权限有效性）**：G9、Path-B 和 retrospective routing 的失败应如何约束自动化权限和 Agent 架构？

### 2.3 五重有效性框架

我们把一个模型主张写成五个必须同时回答的问题：

1. **测量有效性**：字段是否从正确页面和视觉状态读取？
2. **时间有效性**：该字段在拟议决策的 index time 是否已经存在？
3. **构念有效性**：标签是否真的测量要判断的概念？
4. **外推有效性**：评估是否阻断真实 county、operator、template 和 time 结构？
5. **权限有效性**：前四项证据是否足以授权该具体后果？

它们是合取条件，不是可互相补偿的分数。一个页面读得很准的结果态字段仍然可能时间无效；一个 AUC 很高的 shortfall 模型仍然可能构念无效；一个内部可靠的 DN 排序仍然不能自动获得拒绝权限。

### 2.4 评估设计与统计报告

不同实验使用了不同的交叉验证和离散统计口径，本文不把所有“±”解释成同一种不确定性。

- 631 案 historical shortfall 主分析使用按 county 分组的 `StratifiedGroupKFold`：5 folds×5 split seeds。每个 seed 形成覆盖全 cohort 的 out-of-fold（OOF）预测，再对五个 repeat-level 指标报告均值和 SD；例如 0.7400±0.0090 的 0.0090 是五个 split-level AUC 的离散度，不是置信区间。
- Model A 使用 `StratifiedGroupKFold(county)`、5 seeds×5 folds。训练脚本汇总有效 held-out folds 的 AUC；0.8366±0.1097 中的 0.1097 是 fold-AUC dispersion，不是 repeat-level SD 或置信区间。
- Model B 的脚本对五个 seeds 打乱行后调用 `GroupKFold(shuffle=False)`；实际 held-out county sets 未改变。因此把 0.418±0.047 报告为一个有效五折 county-grouped partition 下的 Spearman fold summary，而不是独立 5×5 repeated CV。
- repeat-0 evidence-vs-geography 比较使用同一个 OOF 向量，报告 case bootstrap 和 county-cluster bootstrap；它与五重复均值差不是同一个 estimand。
- G9 使用双侧 Fisher exact test，并按 frozen specification 报告 refused-side cluster collapse 与 tuning-untainted sensitivity。离散检验的 probability-ordered p 值与 equal-tail interval 不保证严格对偶。
- checklist 小样本提供 naive Clopper–Pearson 区间以显示分母不确定性；重复 reader 嵌套于 case 且模型误差相关，因此不作独立 Bernoulli 推断。
- Path-B 使用相同 25 个 grouped folds 的 paired AUC differences，并按预先冻结的 absolute 与 incremental 双门槛裁决。

所有分类结果优先报告 ROC AUC；部分 shortfall 结果还报告 average precision、Brier score 和 top-of-queue precision。回归 proxy 用 Spearman 相关和分档一致率。除明确列出的 bootstrap 或 exact interval 外，SD 均不应读成总体置信区间；多模型探索也未形成覆盖所有比较的 multiplicity-adjusted confirmatory analysis。

### 2.5 术语与内部代号

- **LMM**：large multimodal model，大型多模态模型；用于候选证据抽取或视觉对账。
- **HGB**：histogram gradient boosting，直方图梯度提升；Model A 为 classifier，Model B 为 regressor。
- **OOF**：out-of-fold，只用未参与相应拟合的 held-out predictions 评价。
- **v6**：历史多模态抽取/建模管线的第六次主要修订，不是外部标准版本。
- **G8**：内部盲读可靠性门，用于检查 track separation、词表和字段 emission 的稳定性。
- **G9**：DN 在 audited refused 与 historical approved-status 案之间的冻结关联门；主比较和两个敏感性条件必须同时满足才可开放解释。
- **Path-B**：在旧 `y_needed_short` 任务中加入 checklist features 的冻结 ceiling-break backtest。
- **D1**：相对 TD 的归一化 shortfall；历史 reference 与 current request 使用不同的深度来源，必须分别标注。
- **CK1–CK4**：当前 runtime checklist 的四个 draft items；分别检查 request form、effort narrative/evidence、decision-relevant depth numbers 和 depth-number timeliness。
- **quadrant**：checklist pass/fail × tail yes/no 的 2×2 preliminary triage label；不是审批结论。

## 3. 第一部分：多模态与多代理的证据清理管线

### 3.1 为什么必须多模态

历史案卷包含扫描文本、手写数值、勾选框、图章、井身示意、跨页表格和格式不一致的申请材料。纯文本 OCR 可以帮助检索，却可能丢失“框是否被勾选”“数字属于哪一栏”“印章对应哪份表”等视觉语义。因此本研究把 OCR 定位为候选召回工具，而不是最终事实裁决者：OCR 可以告诉系统“可能在这里”，渲染页面可以否定 OCR-only 解释，审查员才有事实接受权。

v6 管线把 PDF 直接交给多模态接口，并为视觉字段设计来源文件、页码、原文、证据类型和时间/深度轨。后续 full-corpus evidence review 对 635 个源文件目录逐案运行；17 份超大 PDF（14 案）先以 150 dpi 预渲染，以避免传输限制。配置中的模型标识使用浮动 alias，批量请求也未把所有采样参数完全固化，因此本文不把抽取描述为数学意义上的确定性。可复现性来自归档原始响应、派生表和代码版本，而不是假定未来重跑能逐字相同（Fig. 1）。

### 3.2 来源不是附加元数据，而是输出的一部分

一个结构化值只有在能够回答“从哪里读到”时才完整。视觉证据使用如下 provenance tuple：

\[
E=(\text{case},\text{source file},\text{page},\text{quote},\text{visual state},\text{track},\text{timing}).
\]

井属性或部分历史标签来自 legacy Excel register，因此最准确的表述是：每个字段应当能回到原始 PDF 页、legacy register 或确定性派生台账，而不是笼统声称“每个字段都来自 PDF 页”。后续审计中发生的 evidence-pointer 修复说明，字段值正确并不等于来源路径正确；引用准确性必须成为独立评价终点。

### 3.3 严格分离三条深度和时间轨

管线把至少三类状态分开：

- **requested**：operator 在申请时提出或声明的停止深度及理由；
- **approved**：DEP 随后批准或限定的深度；
- **certificate/final-state**：完工后观察到的最终状态、封堵证明或其他结果态信息。

TD anchor、observed obstruction、max reached 和 proposed stop 也分别保留来源与时间属性。任何将 certificate depth 回填成 request-time 输入的做法都会造成时间泄漏；任何将 approved attainable bottom 当作预测特征的做法都可能形成结果态循环。时间轨分离不是数据库美化，而是模型是否在回答前瞻问题的必要条件。

### 3.4 确定性防护、隔离和负向权威

多模态输出之后运行三类 guard：数值合理性、证据/当前作业上下文和时间轨一致性。可疑结果不被悄悄修正或当作缺失值吞掉，而是进入 quarantine 供复核。v6 最终工件包含 10,856 条深度证据行和 436 条隔离记录，并记录 21 个 current-job guard 事件。

这种设计赋予不同组件不同的“否定权”：OCR 可以提出候选，但不能建立事实；视觉页可以推翻 OCR-only 候选；guard 可以阻止可疑字段进入下游；研究者或审查员可以接受、纠正或拒绝字段。多组件并存的价值不在于投票，而在于错误可以沿来源链被定位。

### 3.5 多代理盲读与研究者裁决

在人类领域专家资源稀缺的条件下，项目使用了相互独立运行的 AI-agent 盲读、冻结 emission vocabulary、来源页复核和研究者/用户裁决。其目标是发现定义不清、指针错误和 reader disagreement，而不是用“多个模型都同意”替代专家真值。

主 G8 运行包含 23 案×2 次=46 个 reader outputs，其中 reliability subset 为 18 案×2=36 个 outputs，5 个 anchor 案×2=10 个 outputs 被排除在可靠性分母之外；confirm panel 为 6 案×2=12 个 outputs。不同字段的 11/18、14/18、6/6 和 4/6 是一致性计数，不是正确率。主运行和 confirm 合计的“0/58 contamination”基于嵌套且相关的 outputs，也不能按 58 个独立人类判断计算置信度。

这套内部机制成功促成规则收缩、字段退休、DN 反转和来源修复，但它仍可能具有相关模型误差。本文因此统一称其为“内部独立 AI-agent 盲读与研究者裁决”，不称为“独立人类专家外部验证”。

### 3.6 人类专家何时进入闭环

本研究区分两种 human-in-the-loop：

1. **已经发生的内部人工监督**：研究者核验来源页、裁决部分 disagreement、审阅 DN 阳性和样本化负例；
2. **尚待完成的独立专家验证**：两位领域专家在新的 request-time cohort 上独立编码，第三位专家裁决，并测量类别正确率、深度误差、引用正确率、轨道污染和时间污染。

未来系统应在以下情形强制升级给合格审查员：来源页不可定位；OCR 与视觉状态冲突；requested、approved 和 final-state 轨道冲突；规则后果重大；Model A、Model B、checklist 或 compliance 提示相互不一致；或案例超出训练/参考分布。专家不是在自动决定之后签字，而是在字段获得后果资格之前核验。

### 3.7 管线产生了什么，以及没有证明什么

最终 v6 merge 形成 633 个唯一案级行和 10,856 条深度行。两个大批次的 HTTP 成功计数分别为 818/818 和 483/483；这只是运输完整性，不是案例准确率。在固定的 40 案针对性 regression set 中，observed-depth presence、physical-blocker presence 和 historical-seal presence 三个预定义字段 precision/recall 均为 1.0。由于该集合用于已知失败模式回归，不是总体随机样本，所以结果不能外推为全语料、低质量扫描件或所有字段的准确率。

因此，RQ1 的答案是有限肯定：管线生成了可回溯、可隔离、可审计的证据工件；它尚未建立独立专家参考标准，也尚未证明真实审查中节省时间或降低错误。

### 3.8 历史档案的三块“大陆”

170 列 schema 的语义组成和稀疏性高度结构化，而不是随机均匀缺失（Figs. 2–3）：

- 行政与井属性接近饱和，例如 operator request 96.0%、alternative-method filed 94.8%、coal-seam indicator 90.8%；
- 申请叙述为中等覆盖，例如 request evidence 41.0%、causal reason 36.8%、any struggle 31.8%；
- 深度证据大多约 15%–30%，例如 predecision observed 18.9%、physical blocker 14.6%、historical seal 29.5%；
- 细粒度 work-record difficulty/remedy/obstruction 二值 flags 几乎全部低于 2.5%。

这一区分十分重要。数值型事件计数并不缺失：在 424 案中共有 2,874 条带日期事件，423/424 案至少有一条，中位数为 6 条、最大 30 条。但 2,375/2,874（82.6%）只被归入粗粒度 `other` phase，另有 plugging 457 条、cleanout 42 条。换言之，档案并非“没有作业记录”，而是没有稳定、细粒度且跨文档同构的困难与方法标注。

### 3.9 努力和工具为何不能从低记录率反推

同类困难在申请叙述与独立 work-record aggregation 中的记录率差异显著：cave-in/collapse 为 17.45% 对 0.24%（约 74 倍），hard fill/no progress 为 12.5% 对 2.36%（约 5.3 倍），stuck/lost tools 为 5.90% 对 1.89%（约 3.1 倍），casing failure 为 3.77% 对 1.18%（约 3.2 倍），freshwater-at-depth 为 0.94% 对 0.24%（约 4 倍）；mine-through 为 7.31% 对 0，不能给出有限倍数。

这些是“被记录/被当前抽取规则识别”的差异，不是实际发生率差异。它与申请时点选择性叙述相一致，但不能证明 operator 只在申请时才遇到困难（Fig. 4）。

工具记录更稀疏：197/2,874 条事件含任何工具 token，即 93.15% 没有 token；2,687/2,874（93.49%）的 deployment method 为 unknown。169 个原始 token 包含同义词、粒度混杂和 OCR 噪声，不能解释为 169 种独立工具。工作记录层的 remediation flags 均不超过 0.71%，申请叙述层最高也仅约 1.65%。因此“无工具记录”不能解释成“未使用工具”，当前档案也不足以可靠训练 tool/remedy 模型（Fig. 5）。

上述结果决定了后续模型的权限边界：系统可以收集和提示文档中出现的困难，却不能把文档沉默解释为没有努力，更不能用稀疏工具字段裁决努力充分性。effort-related coverage 的综合图进一步显示，尽管 99.8% 的案例有台账行，明确 attempted-and-failed evidence 只有 6.1%（Fig. 6）。

## 4. 第二部分：如何选择正确的模型——先选择正确的问题

### 4.1 早期 step-location 任务：高分如何由循环标签产生

项目最早试图在每个已批准案例的事件序列中定位历史上被选为最终位置的步骤。最终权威审计集含 103 案、379 条 event rows，并采用 case-grouped 交叉验证。表面上，深度相对 TD 的 gradient boosting AUC 达到 0.916；若使用深度相对 approved attainable bottom，则 GB/RF 可达 0.954/0.958。

但一个不使用任何学习参数的规则——选择每案最深事件——已经在 79/103 案（76.7%）中命中标签。effort-only GB 的 AUC 约 0.717、top-1 命中 64.1%，而相对 approved attainable bottom 的特征直接包含结果态决定。由此产生的正确结论不是“模型从 0.72 提升到 0.96”，而是：

- 0.717 对应有限的过程线索；
- 0.916 主要重现“标签通常是最深事件”的定义结构；
- 0.954–0.958 使用结果态批准锚，属于泄漏。

该 within-case track 因此被退休，不再解释为前瞻审批模型。这是项目第一次问题定义纠错，也是 target audit 的起点：任何复杂模型之前都应先运行零参数 baseline，检查标签是否已经被排序、最大值或结果态字段定义。

### 4.2 RQ2：grouped 泛化下，不同“高分”来自不同任务

项目随后转向 631 案的 case-level historical shortfall proxy。主评估采用 county 为 group 的五折 StratifiedGroupKFold，并以五个 split seeds 重复；这比随机 stratification 更接近面对新地理群组的真实使用情形。

**表 2. 冻结 shortfall cohort 的主要分类结果**

| 评估 | 特征 profile | 模型 | ROC AUC，均值（SD） | 解释 |
|---|---|---|---:|---|
| county-grouped | 非地理 evidence/covariates | HGB | 0.7400 (0.0090) | 内部 shortfall/difficulty ranking |
| county-grouped | v6 predecision depth | HGB | 0.7383 (0.0080) | 更丰富深度证据未提高 AUC |
| county-grouped | legacy 11 | Logistic L2 | 0.7353 (0.0039) | 简单基线仍接近最佳单模型 |

RQ2 的关键不是给 evidence、geography、model class 和 feature richness 分配因果百分比，而是区分四种经验事实：证据/协变量比单独地理更有排序信息；地理分组显著降低乐观估计；非线性在部分比较中有增益；继续堆叠更丰富的 v6 特征没有跨越约 0.74–0.76 的内部 ceiling。性能上限由任务和标签决定，而不只是特征工程。

### 4.3 RQ3：difficulty 不等于 adequacy

历史 shortfall 可以来自困难地质、允许的 attainable-bottom 调整、完成过程、文书规则或真实不足。尤其在 only-approved 或结果态档案中，被观察到的标签具有选择性：成功完成的难案积累更多后续证据，未获批或被补件的案卷则可能停留在不同状态。

项目对 89 个 certificate depth 约等于 approved attainable bottom 的案例进行了探索性删失审计。保留的 ensemble 在原始标签下 AUC 为 0.7567；排除这些可能被批准并完成的案例后为 0.7423；把它们负向重标后为 0.6885。这个敏感性分析不能估计一个新的部署性能，却足以否定较强解释：同一个 historical shortfall label 无法稳定区分“困难但按批准方案完成”与“努力不足”。

因此，约 0.76 的模型被重新命名为 shortfall-risk/difficulty ranking，而不是 adequacy classifier。研究问题从“是否努力充分”转成更窄的两步：先识别有记录的非例行困难，再问当前 proposed shortfall 在相似历史难度参考中是否值得额外解释。

### 4.4 RQ4：checklist 与 DN 的构念收缩

request-package checklist 将事实抽取与规范判断分开，并持续保持 requested、approved 与 certificate 轨道分离。研究 instrument 从 v1 到 v1.4 的演化不是为了美化一致率，而是根据冻结 panel 的失败删除不稳定 token、收窄定义和修复来源指针。当前 runtime 的 CK1–CK4 则标为 **v1.5-DRAFT**，加入 depth-number presence/timeliness 等工程规则，尚未完成法律或独立专家验证，也不应与已完成的 v1.4 内部盲读混称为同一版本。

**表 3. 冻结内部盲读中幸存、修订和退休的 checklist/DN 信号**

| 信号/规则 | 主 panel | confirm/audit | 裁决 | 允许解释 |

DN 被定义为 **documented non-routine difficulty**：材料中有可定位证据表明出现非例行困难。它不等于实际努力、勤勉、合规、充分性、批准或拒绝。424 案中，项目内部逐案裁决 112 个 DN 阳性，其中 108 个 major、4 个 minor；411 个 historical approved-status 案中为 105（25.5%），13 个 audited refused 中为 7（53.8%）。阳性侧逐案内部裁决，负侧只进行 20 案盲样本审计，因此不能称 424 个标签都有独立专家 gold standard。

审计轨迹列出 5 个 substantive positive→negative 反转、1 个 negative→positive 反转和 15 个 evidence-pointer repairs。另一个叙述来源写“6 个 positive→negative”，但只列出 5 案且 taxonomy 也是 5 个；本文采用可枚举的 5，并披露来源不一致。

### 4.5 为什么 108 个档案特征收缩为 17 个申请时点字段

archival DN research model 使用 108 个特征，其中包含 certificate/completion OCR 是否存在等 post-work 变量，因此只能用于档案研究。full-refit permutation importance 中仅 7 项报告为正：最深套管深度 0.00141、alternative-method narrative event count 0.00046、TD 0.00008、causal reason 0.00003、any narrative struggle 0.00002、max observed/TD 0.00001、well age 0.00001；其余 101 项在该次计算中约为零。

这些极小数值只能用于相对排序，不是因果效应，也不等于 101 列字面恒零。108-feature archival HGB 的内部 county-grouped AUC 点估计为 0.8412±0.1136；17-field request-time-compatible Model A 为 0.8366±0.1097，相差约 0.0046。由于没有正式 equivalence test，我们只说二者点估计接近，不说统计等价（Fig. 7）。

Model A 与 Model B 共享以下 17 个字段：

1. TD；
2. well age；
3. casing 数量；
4. deepest casing depth；
5. coal-seam indicator；
6. freshwater indicator；
7. 文件计数；
8. 当前作业事件总数；
9. cleanout 事件数；
10. effort/work 事件数；
11. alternative-method form 是否提交；
12. 是否有 attempted-and-failed 叙述；
13. 是否有 causal reason；
14. max reached depth；
15. obstruction depth/indicator；
16. max reached/TD；
17. 是否填写 proposed stop。

模型不读取 proposed-stop 的具体数值；该数值只在后续 D1 计算中使用。county、operator 和 permit identity 不在这 17 个 serving 字段中。certificate/completion 字段也被排除。所谓 request-time-compatible 仅指字段定义可在申请时收集，不表示它们已经在真实前瞻 intake cohort 上验证。

### 4.6 Model A：DN prescreen HGB classifier——决定“谁先看”

Model A 的训练目标是项目定义下的 DN 标签。它输出 `dn_score`，用于把人工 review queue 从高到低排序。官方内部指标来自 `StratifiedGroupKFold(county)` 5 seeds×5 folds，为 0.8366±0.1097；这里的 SD 是有效 held-out fold AUC 的离散度，不是“83.66% accuracy”、repeat-level SD 或置信区间，也不证明分数是校准概率。

Model A 的作用是稀缺注意力分配：材料更像历史 DN 阳性时，审查员可以更早检查其来源和困难叙述。它不进入 2×2 quadrant 公式，不判断难度档，不判断努力是否充分，也不预测批准概率。然而，队列排序本身仍可能影响等待时间和关注顺序，所以它不是“零后果”的数字；正式使用仍需测量 operator、county、文档质量等群体的排队负担。

### 4.7 Model B：difficulty-proxy HGB regressor——决定“用哪把尺子”

Model B 的 target 不是人工专家标注的客观工程难度，而是冻结 historical shortfall ensemble 的 teacher score。它在 423 个有 teacher 分数的回顾性案例上学习 17 字段到 teacher score 的映射，再按冻结 cutpoints 选择 Q1–Q5 reference bin。

训练脚本虽然循环五个 seeds，但每次只先打乱行，再调用 `GroupKFold(shuffle=False)`；held-out county sets 实际相同。因此 0.418±0.047 应准确描述为**一套五折 county-grouped CV 的 fold-mean Spearman 相关及 fold SD**，而不是 5×5 独立重复 CV，也不是置信区间。Model B 与 teacher 的 exact-bin agreement 为 35.5%，within ±1 bin 为 72.3%；换言之，27.7% 的案例相差至少两档（Fig. 8）。

运行时，Model B 的间接作用如下：

\[
\widehat{s}_{B}(x)\rightarrow \widehat{Q}\in\{1,2,3,4,5\}
\rightarrow P_{90,\widehat{Q}}
\rightarrow \text{tail flag}.
\]

对当前申请，系统另行计算：

\[
D1_{request}=\operatorname{clip}\left(\frac{TD-proposed\ stop}{TD},0,1\right).
\]

若 `D1_request` 超过 Model B 所选 reference bin 的历史 P90，则产生升级旗标；该旗标再与 checklist 联合形成 preliminary quadrant。因此最准确的一句话是：**A 决定谁先看，B 决定用哪把历史尺子；B 不终裁，但选错尺子可能改变 tail 与初始路由。**

### 4.8 “先分组再比较”的证据与尚未解决的迁移假设

在 354 个按代码 operationally defined 的非审计拒绝历史参考记录中，locked teacher score 将案例分为每档 70–71 个。这里的 reference 条件是可评分、`chk_refused_flag=0` 且 TD 非可疑，并未要求所有记录都经过重新确认的 approved status；因此正文不能称其为“354 个确认批准先例”。

teacher-defined Q1→Q5 中，历史 outcome-label shortfall 的中位数从 0.037 增至 0.281，端点约 7.6 倍；DN prevalence 从 8.5% 增至 40.8%，端点约 4.8 倍。这说明 teacher strata 与历史 shortfall、DN 存在明显关联，但不构成独立外部难度真值（Fig. 9）。

更关键的是，历史参考 D1 实际由结果态标签计算：

\[
D1_{ref}=\frac{TD-label\_bottom\_of\_plug}{TD},
\]

而新申请使用 proposed stop。把 `D1_request` 与 `D1_ref` 放在同一经验尺度上，是一个尚未前瞻验证的 transport assumption，而不是已经证明的等价性。

teacher strata 的 P90 分别为 0.332、0.276、0.415、0.456 和 0.552；Q1 的 P90 高于 Q2，也提示有限样本噪声。以 `D1_request=0.45` 为例，它相对五档的示意百分位约为 95.6%、99.5%、91.7%、89.5% 和 73.8%，因而在 Q1–Q3 触发 tail，在 Q4–Q5 不触发。这个例子证明的是条件化比较可以改变“同一个 shortfall 应如何分配审查注意力”，不是证明较困难案例的申请天然合理（Fig. 10）。

### 4.9 pooled threshold 与 conditioned threshold：反事实审查负担，不是公平性证明

若所有参考记录共用 pooled P90=0.455，teacher Q1–Q5 被标记的比例为 4.2%、1.4%、8.6%、11.3% 和 25.4%；Q5 相对 Q2 约为 18 倍。若在每个 teacher bin 内定义 P90，每档约 9.9%–10.0% 被标记。这个结果展示 pooled threshold 会把名义审查量集中在历史困难层，而分层分位数可使 teacher reference 中的 nominal tail mass 接近一致（Fig. 11）。

但“每档约 10%”在很大程度上由 P90 定义保证。更重要的是，图 2–4 使用真实 locked teacher score 分档，而运行时由 Model B proxy 选档；因此这些图是 **teacher-conditioned proof of concept**，不是端到端 Model-B calibration。不能写成“部署系统已经实现各档 10%”“证明公平”或“消除偏差”。真正需要验证的是：在新的 request-time 数据中，经 proxy 选档后 tail rate、错误升级、operator burden 和 reviewer outcome 是否仍可接受。

### 4.10 RQ5：三个失败如何关闭权限

项目没有把负结果放在论文末尾作为免责声明，而是为失败预先规定系统后果。

| G9 tuning-untainted sensitivity | 4/10，OR 1.94，p=0.291 | 要求 supportive p<0.10 | 失败；G9 BLOCKED |
| Path-B | combined AUC 0.6841；相对 auto Δ0.0107（SD 0.0431）；25 folds 中 11 个为正 | combined mean 必须 >0.764 且满足 increment | 未跨越 ceiling |
| anomaly | nominal 10% 下内部 OOF tail 10.17% | 只检查实现/名义尾部 | 不获得 insufficiency 含义 |
| routing smoke | 7 个 depth-related refusal 中 5 个进入 FAST_LANE | 路由方向应符合已知拒绝机制 | final-state retrospective routing 被否证 |

G9 中 3.40 为 cross-product sample odds ratio。固定边际 noncentral-hypergeometric equal-tail 区间包含 1，而 probability-ordered Fisher 双侧 p 值略低于 0.05；离散检验中二者并非严格对偶。由于两个预先冻结敏感性条件失败，不能只取主 p 值重新包装成功。

Path-B 检验的是“把现有 checklist 加到旧 historical shortfall task 是否突破冻结 ceiling”，不是证明任何未来 request-time human label 都无价值。routing smoke 的失败则更直接：结果态 approved/completed 信息使被拒绝案例看起来像成功案例。研究团队没有在看到结果后重调阈值，而是关闭 retrospective approval/refusal routing。

这构成 **failure-gated authority**：

- 早期循环性关闭 step-location 的决策解释；
- 删失审计关闭 shortfall→effort adequacy 的解释；
- G9 失败关闭 DN→refusal/adequacy 的权限；
- Path-B 失败关闭“checklist 已解决旧标签 ceiling”的主张；
- anomaly 只保留 explanation-needed flag；
- routing smoke 失败关闭 final-state 自动加速或不利路由。

## 5. 第三部分：Agent 架构——按证据和权限分层，而不是让多个模型共同审批

### 5.1 “multi-agent”在研究阶段与运行时有不同含义

研究阶段确实使用了多种代理角色：逐案多模态抽取、独立盲读、来源对账、失败复核以及研究者裁决。当前运行时原型则不是一个具有 planner、delegation 和多代理协商的自治系统；它是两个可选 LMM 服务、两个传统 ML 模型、deterministic rules、相似文本检索和人工 review UI 的混合管线。

因此本文使用“多代理证据工程”描述研究与复核分工，使用“混合权限架构”描述 runtime。创新点是 authority separation，而不是 agent 数量。

### 5.2 当前原型的真实接线

当前代码流程可概括为：

```text
PDF → optional LMM prefill → operator/user confirmation
                                  ↓
structured validation → immediate preliminary scoring
    ├─ compliance rules
    ├─ CK1–CK4 checklist
    ├─ Model A → dn_prescreen → review queue order
    ├─ Model B → proxy bin → in-bin P90 tail
    └─ RAG context / supplement hint
                                  ↓
                    preliminary review package
                                  ↓
         post-submit LMM reconciliation (advisory)
                                  ↓
                    reviewer-selected action
```

`/prefill` 最多渲染 30 页 PDF，并生成六个 panel 的候选值及 `{file,page}`；前端把它们标为 suggested，等待确认。结构化提交通过 Pydantic 校验 attestation、深度≤TD、日期、测量方式、失败叙述和未来日期等条件。提交后立即计算 Model A/B、checklist、compliance 和 anomaly，并保存初始 review package。

随后上传文件并排队进行 LMM reconciliation；该 agent 视觉核对 DN claim 的存在、日期、时态和深度，但当前为 Stage-1 advisory，不修改 CK2、原 quadrant 或 Model A 排队。reviewer 最后可以另选 action，Decision 与机器 quadrant 分开存储。

当前 effort-path 的 2×2 公式为：

**表 5. 当前原型的 preliminary 2×2 routing**

| v1.5-DRAFT checklist | Model-B-selected in-bin tail | 当前 preliminary label | 论文允许含义 |
|---|---|---|---|
| pass | no | `FAST_LANE` | 低名义升级优先级；不得直接等同批准 |
| pass | yes | `DEEP_REVIEW` | 材料较完整，但 shortfall 位于所选经验参照档尾部 |
| fail | no | `SUPPLEMENT` | 需要补充 checklist 指定材料 |
| fail | yes | `PUSHBACK` | 缺项且位于 tail；只能触发更高等级人工复核 |

若 compliance rules 为 red，则当前代码覆盖为 `COMPLIANCE_REVIEW`；若 anomaly 不可评分，则只根据 checklist 产生 `FAST_LANE` 或 `SUPPLEMENT`。这些名称是当前软件字符串，其中 `FAST_LANE` 的文案仍带 approval-like 含义，不能被论文当成已获授权的监管行动。

### 5.3 目标架构：先接受证据，再允许后果

根据失败结果，目标架构应把顺序改成（Fig. 12）：

```text
request-time structured intake
→ claim-to-source reconciliation
→ qualified reviewer accepts relevant fields
→ bounded ML / checklist / compliance / retrieval
→ non-decisional review routing
→ human decision + mandatory reasons
→ versioned prospective labels
```

其核心不是在末端增加一个“human approval”按钮，而是把人工接受前移：任何不利、加速或可能改变审查强度的 route，只有在合格审查员打开来源页、核验轨道并主动接受相关字段后才可生效。

### 5.4 各组件的最小权限

| 组件 | 被允许的职责 | 当前是否影响 preliminary quadrant | 禁止解释 |
|---|---|---:|---|
| Prefill LMM | 提出可编辑字段和来源页候选 | 用户接受后可间接影响 | 自动建立事实或审批 |
| Reconciliation LMM | 输出 verified/mismatch/not_found 等对账旗标 | 否，当前 advisory | 已具 veto 或已通过外部盲评 |
| Model A | DN prescreen；排序人工队列 | 不进入 quadrant | 努力充分性、难度、批准概率 |
| Model B | 近似 teacher score；选择 peer bin | 经 tail 间接进入 | 客观难度、努力分数、批准概率 |
| Historical anomaly table | 比较 current request D1 与所选参考档 | 是，作为 tail 维度 | 违规、失当、应拒绝 |
| CK1–CK4 | 提示材料完整性、时效和数字冲突 | 当前直接进入 | 完整法律标准或未经核验的自动 veto |
| Compliance rules | 对三个声明字段产生 red flag | 当前覆盖 quadrant | 查全率已验证的自动法律裁决 |
| RAG | 返回相似文本上下文 | 不直接改变 quadrant | 已验证 precedent、批准可能性或模型推理 |
| Human reviewer | 核验来源、解释规则、选择最终行动 | 最终行动主体 | 把责任委托给模型 |

### 5.5 当前原型与目标架构之间尚未闭合的缺口

代码审计显示，下列能力仍应写为未来要求，而不是已实现保障：

1. 当前 quadrant 在来源对账和 reviewer acceptance 之前计算并显示；它只能称 preliminary triage label。
2. reconciliation 尚未真正 gate CK2 或 route，预设的未来开关未形成完整执行路径。
3. 前端发送的 `field_provenance` 尚未由当前 API schema 完整持久化，来源接受闭环未完成。
4. `file_names` 可以是声明值或占位符，不能保证后台实际存在相应 UploadedFile；文件计数和 CK2 可能被弱来源影响。
5. compliance red 目前覆盖 quadrant，却没有真正停止或隐藏 Model A/B、anomaly outputs。
6. lane 文案仍含 `Recommend approval` 等决策语义，与“lane 不应暗示审批”的目标不一致。
7. reviewer 可以另选 action，但没有逐字段 accept/reject、bin override、mandatory reason、证据快照及模型/规则版本强制固化。
8. decision log 是 structured prototype feedback，不是已经验证的 clean gold labels。
9. RAG 的 20-query validation 未完成，metadata 和 filtering 仍有缺陷；“verified precedents”措辞过强。
10. 部分 LMM/PDF runtime 依赖和真实 PDF 端到端验证尚未完整进入 deployment requirements。

公开这些差距不是削弱论文，而是本文方法论的直接实践：系统能够运行某段代码，不等于该段代码已获得相应制度权限。

### 5.6 不确定案例的 expert-in-the-loop 策略

未来可以把人工稀缺资源集中到五类案例：

- 来源不存在、页码冲突或视觉状态无法确认；
- Model B 位于 bin 边界或与 reviewer 选档分歧；
- checklist 失败但文档语境复杂；
- compliance red 可能产生不利后果；
- Model A 排名、Model B tail、规则和相似案例给出相互矛盾的信号。

这些案例应由合格领域审查员完成字段级 acceptance/rejection、必要的 bin override、最终 action 和 mandatory reason。系统保存 index-time evidence snapshot、规则版本、模型版本、override 及来源页，才能逐步形成未来可训练的、真正前瞻性标签。

## 6. 讨论

### 6.1 不同 AUC 不是同一模型的连续进步

项目中出现过约 0.916、0.759、0.841 和 0.837 等数字。它们属于不同 unit、index time、target、feature window 和 allowed action：0.916 是早期循环性 step-location；约 0.759 是 historical shortfall ensemble；0.841 是含 post-work 字段的 archival DN research model；0.837 是 17-field request-time-compatible DN prescreen。把它们写成“0.72→0.92→0.84 的模型升级”是类别错误。

模型成绩必须与五元组一起报告：

\[
(\text{unit},\text{index time},\text{target},\text{evidence window},\text{permitted action}).
\]

没有这个五元组，AUC 不是科学结论，只是一个脱离问题定义的数。

### 6.2 负结果不是附录，而是架构输入

本项目最有价值的结果包括：复杂特征没有突破 ceiling；新增案例没有提高 grouped AUC 点估计；拒绝 ground truth 在来源审计后坍缩；G9 敏感性失败；Path-B 未过门槛；结果态 routing 方向相反。每一次失败都撤销一种权限，而不是触发事后更换阈值。

这种方法可以称为 failure-gated authority。它比“先部署、再在 limitation 中说明风险”更强，因为失败的后果在模型输出获得制度含义之前发生。

### 6.3 来源追踪应当成为模型评价终点

文档 AI 的准确性至少包含字段值、引用、轨道和时间四个维度。一个数值即使正确，如果引用了错误页或来自完工证明，它仍不适合作为 request-time evidence。未来评价应分别报告：categorical accuracy；深度 exact/±25/±50 ft；citation correctness；requested/approved/final-state contamination；index-time contamination。

### 6.4 difficulty conditioning 是审查量设计，不是规范裁决

对不同 teacher strata 使用不同经验 P90，解决的是“不要让困难层因为共用阈值而承担不成比例的 nominal review burden”这一操作问题。它不证明历史批准都合理，也不定义法律上的 reasonable effort。特别是历史 reference 使用 outcome-label shortfall，而新申请使用 proposed shortfall；这个迁移必须在前瞻 cohort 中验证。

Model B 的作用也应理解为参照选择，而不是难度真值估计。exact-bin agreement 仅 35.5%，说明其输出不足以支持不透明的自动后果。未来可以把连续 uncertainty、相邻档敏感性和 reviewer override 同时展示；若相邻档 P90 会改变 flag，则自动升级而不是静默选择。

### 6.5 hybrid accountability 不是“模型加规则”

模型擅长排序和近似，规则擅长表达明确、可审查的条件，LMM 擅长从异构页面提出候选，人类擅长解释语境和承担规范责任。混合架构的关键不是把四者相加，而是确保每一层不能越权：

- LMM 有 proposal/recall authority，无 factual authority；
- Model A 有 attention-allocation authority，无 adequacy authority；
- Model B 有 reference-selection authority，无 objective-difficulty authority；
- deterministic rules 有 visible-flag authority，无完整法律裁决权；
- 人类审查员拥有 evidence-acceptance 和 decision authority。

### 6.6 可迁移性

该框架可迁移到许可审查、环境修复、职业安全、福利裁定和基础设施检查等文档密集场景，但具体模型、词表和清单不能直接迁移。每个场景都必须重新确定构念、index time、来源类型、真实 group structure 和 allowed action。

## 7. 局限性

第一，最终 checklist、DN 和 refusal mechanisms 没有独立领域专家外部 gold standard。相关 AI-agent 一致性可能共享系统误差。

第二，同一 635 案文档宇宙参与了失败发现、schema 修订、阈值形成、DN coding、模型训练、anomaly 分析和原型构建。Git 内冻结减轻了部分适应性复用，但不是完全 untouched external test。

第三，county-grouped CV 没有阻断 operator、关联井、重复 template、form revision 或 time。Model B 的五个 seed 也没有形成独立重复 folds；其 ±0.047 只是五折 fold SD。

第四，277/912 个无本地 source PDF 的目录可能与年龄、operator、county、复杂度或 outcome 系统相关，数字可得性本身可能选择样本。

第五，拒绝案例仅 13 个且机制异质；任何 nominal p 值和 post-hoc 机制对比都高度脆弱。fold SD 也不是 population confidence interval。

第六，历史 reference 的 354 案是 operationally defined non-refused set，并非全部经重新核实的 approved precedents。其 D1 使用结果态 `label_bottom_of_plug`，与新申请 `proposed_stop` 的可比性尚未验证。

第七，Model A 的 DN 标签和 Model B 的 teacher 均来自同一回顾性档案；request-time-compatible feature definition 不等于 request-time external validation。

第八，historical 3-category phase 与 production 10-category EVENT_TYPES 缺少专家 crosswalk；历史 `n_effort_work_events` 与 intake `n_effort_events` 语义不完全同构（Fig. 13）。

第九，模型 alias、部分 runtime 参数和依赖未完全固定；归档响应提高可审计性，但不能保证未来逐字复现。

第十，当前没有前瞻 reviewer study，因此没有证据证明系统缩短审查时间、提高一致性、改善环境结果或避免 automation bias。

## 8. 前瞻验证方案

下一阶段应先验证数据和人机接口，再考虑模型再训练。

### 8.1 新的 request-time reference cohort

只保留申请提交时已经存在的表单、附件和字段；冻结 index time，禁止 certificate/final-state 回填。按 operator、time、template family 和 county 建立外部留出，并预先登记 eligibility、missing-data、threshold 和 model-selection 规则。

### 8.2 独立专家参考标准

由两位油气井/DEP 领域专家独立编码 primary fields、DN、request completeness、difficulty stratum 和 refusal mechanism；第三位专家裁决。分别报告 disagreement，而不是只报告 adjudicated labels。

### 8.3 端到端 Model B 验证

在真实 intake 中验证 teacher-to-proxy transport、exact/adjacent-bin agreement、bin-boundary uncertainty、proxy-conditioned tail calibration 和 subgroup review burden。必须把 teacher-conditioned 图示与实际 proxy path 分开评估。

### 8.4 前瞻性人机 crossover

比较四种条件：unaided review；evidence retrieval only；evidence+checklist；full bounded interface。终点包括 review time、漏证据、专家一致性、override rate、对故意错误建议的恢复、confidence calibration、operator contestability 和 differential burden。

### 8.5 权限开启标准

每一项新 authority 都应有单独 gate。证据提取通过不自动开启 routing；routing utility 通过不自动开启 adverse recommendation；任何 approval/refusal 辅助都需要独立法律、工程、差异影响和 automation-bias 审查。

## 9. 可复现性、伦理与治理

项目保存了源表、原始 API 响应、派生工件、protocol files、out-of-fold 结果和版本历史。历史协议在本地 Git 中先于相应运行冻结，但没有证明事先存入不可变第三方 registry，因此本文使用“version-controlled prospective protocol freezing”，不使用 formal preregistration。

为实现独立复核，应冻结完整分析环境和 artifact hashes；固定模型标识、prompt、rendering 参数和 fold assignments；发布去标识化 data dictionary、model card、intended/prohibited uses 和 claim-evidence ledger；在 DOI-bearing repository 中存档可公开工件。行政 PDF 可能包含姓名、签名、电话和电子邮件，公开前需进行正式 disclosure review。

当前记录没有形成 IRB determination。任何前瞻 reviewer/operator study 均应在开始前取得 IRB 或 non-human-subject determination。系统还需要 authentication、RBAC、retention/deletion policy、incident response、字段级审计日志、模型/规则版本快照和 operator correction/appeal 机制。

AI 系统参与了抽取、代码开发、历史重建、内部盲读和稿件起草。AI 不能成为作者或承担责任；人类作者必须逐项复核分析、按期刊政策披露辅助并对最终主张负责。

## 10. 结论

历史监管档案可以让模型准确地回答错误的问题。本研究中，早期高分主要重现循环标签；case-level 模型识别的是 historical shortfall/difficulty，而不是 effort adequacy；结果态档案还把已知深度型拒绝错误送入快线。正确的响应不是继续无约束增加特征，而是把证据、时间、构念、外推和权限分开，并允许失败撤销权限。

最终形成的不是自动审批器，而是一套受限的证据与注意力分配架构：多模态代理提出带来源候选；Model A 说明谁应先被人工查看；Model B 提议用哪一档历史尺子；checklist 和 rules 使缺项与冲突可见；审查员核验来源并承担最终行动。当前原型已经能够生成研究型 review package，却尚未强制实现字段级人工接受、完整 provenance 闭环或前瞻性效用验证。

因此，本文最重要的成果不是某个最高 AUC，而是一种可复现的研究纪律：**先证明模型有资格回答这个问题，再讨论它回答得有多准；先确定每个组件被允许做什么，再讨论如何组合它们。**
