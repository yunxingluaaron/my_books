# Draft 重写说明与论文结构

## 1. 本轮重写的中心论点

这不是一篇宣称“我们造出了自动审批器”的论文，而是一篇研究 Agent 时代为何会暴露历史数据库的 decision-trace 缺口，以及高后果档案 AI 应如何在无法弥补该缺口时收缩权限的论文。全文的中心论点是：

> **许多机构不缺结果数据，而缺少可回放的决策痕迹。模型性能不能从 final state 还原 decision state；只有当证据、时间、构念、外推和权限五个条件同时成立时，模型分数才可能获得决策含义。**

论文按照用户指定的三条主线重组：

1. **多模态与多代理证据清理管线**：说明异构 PDF 中散落的 request、work、correspondence、approval 和 certificate fragments 如何被重建为带文件、页码、原文、视觉状态和时间轨的 partial decision traces；多个模型代理、确定性规则与研究者裁决如何分工；以及为什么内部盲读不等于独立领域专家金标准。
2. **如何选择正确的模型与正确的任务**：从早期 step-location 的循环标签，经县级分组泛化和删失审计，证明 final-state shortfall 不能替代 request-time effort adequacy；再把“努力证据痕迹”和“难度情境”分开，说明 Model A 只负责 DN 排队，Model B 只负责粗粒度难度条件化，二者均无审批权限。
3. **受限权限的 Agent 架构**：不是让 Agent 从结果表猜回原因，而是在新的 intake 中原生记录 decision state；候选证据生成、来源核验、规则检查、模型排序、同类异常提示和最终决策被拆成不同权限层，任何不利或加速后果都必须在合格审查员核验并主动接受相关字段后才能生效。

## 4. 人类参与的准确表述

历史项目使用了研究者/用户裁决、来源页复核和少量人工确认，也使用了相互独立运行的 AI 代理盲读。它们构成内部质量控制，但不构成两个独立领域专家建立的外部参考标准。因此正文统一使用：

> “独立 AI-agent 盲读与研究者裁决的内部验证”

而不使用：

> “独立人类专家验证已经完成”

真正的 human-expert-in-the-loop 是下一阶段设计要求：对高不确定、来源冲突、规则后果重大或模型与规则不一致的个案，由合格领域审查员核验；前瞻性研究还需要两位独立领域专家编码、第三位专家裁决。

## 5. Decision trace 与“两个证据轴”的准确表述

历史案卷并非完全没有决策信息，而是把它们以碎片形式散落在申请书、作业记录、往来函件和结果文书中。论文将 decision trace 定义为一个可回放单元：

> `case + index time + actors + observed state + request/action + attempted alternatives + source evidence + applicable rule/version + decision/reason + uncertainty/override + later outcome`

其中 later outcome 必须与 decision-time state 分离。现有数据最多支持重建 partial traces，不能把缺失部分由模型补写成事实。

“reasonable effort”可以由两条证据轴支持审查，但不能被它们自动测量：

1. **努力证据痕迹**：何时尝试了什么、是否失败、使用了什么工具或补救方法、到达何深度、原因和来源在哪里。工具只能在有可靠记录时使用；93.15% 的事件无工具 token，所以 absence 不是 no effort。
2. **难度情境**：井身和物理条件，例如 TD、井龄、套管、煤层、淡水、障碍与到达深度；county/formation geography prior 是另一种群组背景，不能与这些非地理井属性混称。

最终 adequacy 是在适用时点的规则、被接受的努力证据、难度情境和合格审查员判断下形成的规范性结论。25 Pa. Code §78.1 的正式措辞是 **reasonable effort**，不是 maximum/greatest effort。

## 6. 两个模型的一句话分工

- **Model A（DN prescreen HGB classifier）**：回答“哪一份材料更值得先由人做 documented-difficulty 深度复核？”输出 `dn_score`，只用于队列排序；DN 不是 effort trace completeness 或 reasonable-effort adequacy。
- **Model B（difficulty-proxy HGB regressor）**：回答“这份申请应当与哪一档 operationally defined 历史参照记录比较？”输出难度代理值并选择 peer bin；随后才计算档内异常百分位。它不使用 county/formation geography-only profile，也不产生客观难度真值；参照集并非全部经重新确认的批准先例。

Model A 不决定四象限路由，Model B 不预测审批；规则可以提出合规复核，清单可以提示缺项，异常只能提示解释需要，最后行动由人负责。
